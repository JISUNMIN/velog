객체를 배열로 변환할 때 자주 사용하는 `Object.entries()`, `Object.keys()`, `Object.values()` 메서드의 차이점과 활용법에 대해 정리해보겠습니다.



---

## 🧩 기본 문법
>```ts
Object.entries(obj: object): [string, any][]
Object.keys(obj: object): string[]
Object.values(obj: object): any[]
```

>JavaScript 객체의 키(key)는 항상 `문자열(string)` 타입으로 처리됩니다.



| 메서드             | 반환값 형태         | 주요 내용 및 사용처                         |
|--------------------|---------------------|-------------------------------------------|
| Object.entries(obj) | 2차원 배열           | `[key, value]` 쌍 배열, 키-값 쌍 같이 필요할 때 사용 |
| Object.keys(obj)    | 1차원 배열           | 키(key)만 배열로 반환, 키 목록이 필요할 때 사용     |
| Object.values(obj)  | 1차원 배열           | 값(value)만 배열로 반환, 값 목록이나 값 기준 연산 시 사용 |

## 주요 활용 예시

### 1. Object.entries()
키와 값을 함께 다뤄야 할 때 사용합니다.

- 배열 메서드(`map`, `filter`, `findIndex`)와 함께 객체를 탐색할 때 유용합니다.
- 객체를 배열 형태로 변환하여 반복 작업이 필요할 때 활용합니다.

```js
Object.entries(obj).forEach(([key, value]) => {
  console.log(`${key}: ${value}`);
});
```

### 2. Object.keys()
객체의 모든 키 목록을 얻고 싶을 때 사용합니다.

- 키 목록만 필요한 경우(예: 폼 필드명, 설정명 등)에 활용합니다.

```js
const keys = Object.keys(obj);
keys.forEach(key => console.log(key));
```

### 3. Object.values()
값 목록만 필요할 때 사용합니다.

- 값에 대한 조건 검사, 합산, 필터링 등 연산 시 활용합니다.

```js
const values = Object.values(obj);
const sum = values.reduce((acc, val) => acc + val, 0);
```

## ✅ 요약 정리
- Object.entries()는** 2차원 배열을 반환**합니다.
- Object.keys()와 Object.values()는 **1차원 배열을 반환**합니다.
- 객체를 다룰 때 필요한 정보(키, 값, 둘 다)에 맞춰 메서드를 선택해 사용해야 합니다.
- 배열 메서드와 조합하여 유연하게 데이터를 처리할 수 있습니다.
