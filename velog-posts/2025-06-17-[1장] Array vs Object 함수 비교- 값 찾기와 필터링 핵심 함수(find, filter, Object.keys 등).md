### 1. 배열 함수 정리  

#### 1-1. `find()`  
- 설명: 조건에 맞는 **첫 번째** 요소를 반환, 이후 조건에 맞는 요소는 찾지 않음.  
- 조건에 맞는 값이 없을 시: `undefined`  
- 예시:  
  ```js
  const arr = [1, 2, 3, 4];
  const found = arr.find(x => x > 2); // 3
```

#### 1-2. `filter()`
- **설명**: 조건에 맞는 **모든 요소**를 배열(얕은 복사본)로 반환 
- 조건에 맞는 값이 없을 시: `[]` 
- 예시:
  ```js
  const arr = [1, 2, 3, 4];
  const filtered = arr.filter(x => x > 2); // [3, 4]
```

```js
const value = [1, 2, 3];
const userId = 2;

```

#### 1-3. `some()`
- **설명**: 조건을 만족하는 요소가 하나라도 있으면 `true` 반환  
- 조건에 맞는 값이 없을 시: `false` 
- 예시:
  ```js
  const arr = [1, 2, 3, 4];
  const hasLarge = arr.some(x => x > 3); // true
```

#### 1-4. every()
- **설명**: 모든 요소가 조건을 만족하면 `true` 반환
- 조건에 맞는 값이 없을 시: `false` 
- 예시:
  ```js
  const arr = [1, 2, 3, 4];
  const allLarge = arr.every(x => x > 0); // true
```


### 주요 배열 탐색 함수 비교 및 활용 가이드

- `find()`는 조건에 맞는 **첫 번째** 요소를 반환하며, 없으면 `undefined`를 반환합니다. 단일 요소 검색에 적합합니다.  
- `filter()`는 조건에 맞는 **모든 요소**를 모은 새 배열을 반환하며, 조건에 맞는 요소가 없으면 빈 배열(`[]`)이 나옵니다. 여러 요소를 걸러낼 때 사용합니다.  
- `some()`은 조건을 만족하는 요소가 **하나라도 있으면** `true`, 없으면 `false`를 반환합니다. 조건 충족 여부 확인에 편리합니다.  
- `every()`는 모든 요소가 조건을 만족해야 `true`를 반환하며, 하나라도 조건을 만족하지 않으면 `false`입니다. 모든 요소 검사에 적합합니다.  

** ✅ 실무에서는**  

| 구분         | 함수 이름      | 반환값 종류       | 동작 방식                          | 활용 예시                   |
|------------|------------|----------------|-------------------------------|--------------------------|
| **값 반환**    | `find()`    | 단일 요소 | 조건에 맞는 **첫 번째** 요소 반환          | 단일 값 검색 시 사용             |
|            | `filter()`  | 배열 (얕은 복사본)    | 조건에 맞는 **모든 요소** 배열로 반환       | 여러 값 필터링 시 사용            |
| **조건 충족 여부 확인** | `some()`    | `boolean`        | 조건에 맞는 요소가 **하나라도 있으면** `true` | 조건 중 일부라도 만족하는지 여부 확인      |
|            | `every()`   | `boolean`        |  ** 모든 요소가 조건을 만족 ** 해야 `true`        | 모든 조건을 충족하는지 여부 확인          |

### 2. 객체 함수 정리 

#### 2-1. Object.keys()
- 설명: 객체의 **모든 키**를 **배열**로 반환
- 조건에 맞는 값이 없을 시: `[]` 
- 예시:
  ```js
  const obj = { a: 1, b: 2 };
  Object.keys(obj); // ['a', 'b']
```

#### 2-2. Object.values()
- **설명:** 객체의 **모든 값**을 **배열**로 반환
- 조건에 맞는 값이 없을 시: `[]` 
- 예시:
```js
const obj = { a: 1, b: 2 };
Object.values(obj); // [1, 2]
```

#### 2-3. Object.entries()
- **설명:** 객체의 **키-값 쌍을 배열 `[key, value]` 형태**로 반환

- 예시:
```js
const obj = { a: 1, b: 2 };
Object.entries(obj); // [['a', 1], ['b', 2]]
```
### 주요 배열 탐색 함수 비교 및 활용 가이드
- **키, 값, 쌍** 중 필요한 정보에 따라 적절한 함수 선택합니다.
- 객체 관련 함수들은 **객체를 배열 형태로 바꿔주는 역할**을 해주기 떄문에,
그 이후에는 보통 map(), filter(), forEach(), reduce() 같은 **배열 메서드들과 조합해서 활용**합니다.
- **객체를 배열처럼** 다룰 때 `Object.entries()` + 배열 함수 조합 활용  
- `Object.keys()`는 객체의 **키 배열**이 필요할 때 사용합니다.  
  예: 키 존재 여부 확인, 동적 렌더링 시 키 반복  
- `Object.values()`는 객체의 **값 배열**이 필요할 때 사용합니다.  
  예: 값 중 특정 조건 검사, 합산 등  
- `Object.entries()`는 객체를 **키-값 쌍 배열**로 다룰 때 사용합니다.  
  예: 필터링 후 객체 재구성, key-value 출력용 순회, API 쿼리 문자열 생성 등  

** ✅ 실무에서는**

| 구분               | 함수 이름       | 반환값 종류      | 동작 방식                                   | 활용 예시                                  |
|------------------|--------------|--------------|----------------------------------------|---------------------------------------|
| **키 반환**          | `Object.keys()`   | 배열           | 객체의 모든 **키**를 배열로 반환                     | 키 존재 여부 확인, 동적 렌더링 시 키 반복            |
| **값 반환**          | `Object.values()` | 배열           | 객체의 모든 **값**을 배열로 반환                     | **값 조건 검사**, 합산 등 값만 필요할 때 사용             |
| **키-값 쌍 반환**      | `Object.entries()`| 배열           | 객체의 **키-값 쌍**을 `[key, value]` 배열 형태로 반환 | 필터링 후 객체 재구성, key-value 출력, API 쿼리 문자열 생성 등 |
 
>❓ 1. 객체를 배열로 바꾸는 이유? 순회하고 조작하기 쉬워서 

| 이유                         | 설명                                                                                                           |
| -------------------------- | ------------------------------------------------------------------------------------------------------------ |
| 🔁 **반복문 사용이 간편함**         | 객체는 `for...in` 외에 순회가 까다로워. 반면 배열로 바꾸면 `map`, `filter`, `reduce`, `forEach` 등 배열 메서드를 쓸 수 있어서 코드가 간결하고 직관적임. |
| 🔍 **조건 필터링이나 검색이 쉬움**     | `Object.entries(obj).filter(([key, value]) => value > 10)`처럼 조건 걸기가 배열이 훨씬 편해. 객체는 이런 연산이 바로 안 됨.            |
| 🛠️ **데이터 가공/변환이 쉬움**      | 배열 메서드들은 데이터를 재구성하는 데 최적화돼 있어서, 객체의 내용을 재구성하거나 새로운 형태로 만들 때 유용함.                                             |
| 📦 **프론트에서 데이터 렌더링과 잘 맞음** | 배열은 컴포넌트에서 리스트 렌더링(`.map`)할 때도 쓰이니까, 객체 데이터를 배열로 바꿔야 바로 렌더링 가능함.                                             |



>❓ 2. 값 조건 검사는 배열로 뽑은 값들 중에서 특정 조건에 맞는 게 있는지 확인하는 것.
예를 들어, 점수 배열에서 80점 이상인 값이 있는지 체크할 떄
```
예시) 
const scores = { math: 75, english: 82, science: 59 };
const allPassed = Object.values(scores).every(score => score >= 60);
console.log(allPassed); // false (science가 59점이라서)
````

###  Array와 Object의 비슷한 함수끼리 묶어 비교하기

| 기능       | 배열 메서드              | 객체 메서드                               | 설명                  |
| -------- | ------------------- | ------------------------------------ | ------------------- |
| 키 조회     | -                   | `Object.keys()`                      | 객체의 키 리스트 조회        |
| 값 조회     | -                   | `Object.values()`                    | 객체의 값 리스트 조회        |
| 키-값 쌍 조회 | -                   | `Object.entries()`                   | 객체의 키-값 쌍 배열 반환     |
| 조건 검색    | `find()`            | `Object.entries().find()`            | 조건에 맞는 요소/키-값 쌍 찾기  |
| 조건 필터링   | `filter()`          | `Object.entries().filter()`          | 조건에 맞는 요소/키-값 쌍 필터링 |
| 조건 검사    | `some()`, `every()` | `Object.entries().some()`, `every()` | 조건 만족 여부 검사         |
