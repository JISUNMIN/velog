## 키가 변수일 때는 왜 점 표기법이 안 될까요?

---

### ❌ 잘못된 방법 (점 표기법으로 변수 접근하려고 할 때)

```js
const key = 'name';

const user = {
  name: '다람쥐',
};

// 이렇게 하면 안 됩니다!
console.log(user.key); // undefined


```

❓ 왜 안 되는 걸까?
user.key는 "key"라는 글자 그대로의 속성을 찾으려는 것입니다.
하지만 user 객체에 "key"라는 속성은 없습니다.
우리가 원하는 건 "name"이라는 변수의 값을 키로 쓰는것 입니다.

✅ 올바른 예시 (대괄호 표기법)
```js
const key = 'name';

const user = {
  name: '다람쥐',
};

// ✅ 변수로 동적으로 접근하려면 대괄호 사용!
console.log(user[key]); // '다람쥐'

```

🔁 비교 요약

| 코드          | 설명                            | 결과          |
| ----------- | ----------------------------- | ----------- |
| `user.key`  | `"key"`라는 이름의 속성을 찾음          | `undefined` |
| `user[key]` | `key` 변수의 값 (`'name'`)을 키로 사용 | `'다람쥐'`      |


## ✅ 요약 정리
👉 "**변수로 키에 동적으로 접근하고 싶다면 [] (대괄호 표기법)을 사용해야 합니다.**"
**점(.) 표기법은 고정된 키** 이름에만 사용할 수 있습니다.
