Next.js 프로젝트에서 화면 내 특정 요소(예: 로고)를 클릭하여 페이지를 이동할 때, `Link` 컴포넌트를 사용하는 것이 권장되는 이유와 실제 구현 방법을 정리했습니다.  
원래는 로고 클릭 시 `router.push()`를 사용해 페이지 이동을 처리했지만, Next.js 공식 문서와 커뮤니티에서 `Link` 컴포넌트를 사용하는 방식을 더 권장한다는 점을 알게 되어,  
왜 `Link`가 더 좋은지와 실제 적용 한 것을 정리했습니다.


## ✅ Link를 사용하는 이유

### 1. **Next.js가 자동으로 최적화를 해줍니다**

- `Link`를 사용하면 Next.js가 해당 페이지를 미리 **프리페치(prefetch)** 합니다.
- 사용자가 마우스를 올려두기만 해도 Next.js는 그 페이지의 JS 코드와 데이터를 미리 불러오기 시작합니다.
- 결과적으로 페이지 전환 속도가 훨씬 빨라집니다.

### 2. **접근성(Accessibility)이 뛰어납니다**

- `Link`는 HTML의 `<a>` 태그로 렌더링되기 때문에,
- 키보드 탐색 (Tab 키)과 스크린 리더 같은 보조기술에서 올바르게 작동합니다.
- 반면, `div`나 `img`에 `onClick` 이벤트만 붙인 방식은 접근성이 떨어지고, 별도의 키보드 접근 처리 없이는 접근하기 어렵습니다.

### 3. **표준에 맞는 의미 있는 시맨틱 구조를 제공합니다**

- 로고 클릭 시 페이지 이동은 **링크 역할**이므로, 의미상 `<a>` 태그를 사용하는 것이 올바릅니다.
- 시맨틱한 구조는 유지보수와 SEO에도 긍정적인 영향을 미칩니다.

>SEO = Search Engine Optimization 검색엔진 최적화

---

## 🔁 `router.push()` + `onClick` 방식은 언제 사용할까요?

- 버튼 클릭 시 특정 로직 처리 후 조건부로 이동해야 할 때 사용합니다.
- 혹은 복잡한 커스텀 이벤트 로직이 포함된 경우에 적합합니다.

```tsx
if (isLoggedIn) router.push("/dashboard");
```

## ✍️ 실제 구현 예시: Link로 로고 감싸기

```

import Link from "next/link";
import Image from "next/image";
import { logo } from "@/assets/images";

export function LogoArea() {
  return (
    <div className="flex flex-col items-center">
      <Link href="/projectlist">
        <Image
          src={logo}
          alt="Logo"
          priority
          className="cursor-pointer w-[100px] md:w-[120px] lg:w-[150px]"
        />
      </Link>
      <p className="text-base sm:text-lg lg:text-2xl font-extrabold text-center text-gray-800 drop-shadow-md mb-2">
        Squirrel Board
      </p>
    </div>
  );
}
```

## ✅ 요약

| 방식                        | 장점                               | 단점                       |
| ------------------------- | -------------------------------- | ------------------------ |
| `Link` 컴포넌트 사용            | 빠른 페이지 전환, 프리페치, 뛰어난 접근성, 시맨틱 구조 | 복잡한 조건부 로직 처리에는 불편할 수 있음 |
| `router.push()` + onClick | 조건부 이동 로직 구현 가능                  | 프리페치 불가, 접근성 떨어짐         |


## 결론
단순히 “로고 클릭 → 페이지 이동” 기능만 구현할 경우에는,
Link 컴포넌트를 사용하는 것이 Next.js 친화적이며 사용자 경험 측면에서도 가장 좋은 방법입니다.
이번에 공부하면서 기존에 쓰던 router.push() 방식보다 더 권장되는 이유를 알게 되어 실제 프로젝트에 적용했습니다.
