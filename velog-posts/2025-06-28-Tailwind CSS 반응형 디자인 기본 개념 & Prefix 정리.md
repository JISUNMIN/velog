##  📱 핵심 개념: 모바일 우선 (Mobile First)
모바일 우선 설계는 널리 권장되는 반응형 디자인 방식입니다.  
기본 스타일을 모바일 화면 크기에 맞추고, 점점 큰 화면에서는 추가 스타일을 덮어쓰는 방식입니다.

Tailwind는 기본 스타일이 모바일 기준이고,  
`sm:` 같은 prefix는 해당 사이즈부터 스타일을 덮어씀을 의미합니다.

예시:
```tsx
<div className="bg-red-500 sm:bg-blue-500" />
```

- 모바일 (~639px): bg-red-500
- 640px 이상: bg-blue-500

즉, sm:은 모바일 너비를 제외한 그 이상의 화면 크기에서 모두 적용됩니다.

## Tailwind CSS 반응형(prefix) 정리


| Prefix | 적용 기준 화면 너비 (min-width) | 의미                  |
| ------ | ------------------------------ | --------------------- |
| sm     | 640px 이상                     | 작은 태블릿 이상 (모바일 아님) |
| md     | 768px 이상                     | 일반 태블릿 이상      |
| lg     | 1024px 이상                    | 작은 노트북 이상      |
| xl     | 1280px 이상                    | 데스크탑 이상         |
| 2xl    | 1536px 이상                    | 큰 데스크탑 이상      |

근데 Tailwind의 prefix가 이렇게 많다 보니,  
실무에서 모든 사이즈를 다 쓰기보다는  
아래 세 가지 prefix를 가장 많이 사용합니다.  

---

## 🔥 실무에서 가장 많이 쓰는 3가지 prefix

| Prefix | 화면 크기 기준  | 실무에서 주 사용처               |
| ------ | --------------- | ------------------------------- |
| sm     | 640px 이상      | 모바일 → 태블릿 전환점 (기본)   |
| md     | 768px 이상      | 태블릿 → 작은 노트북 전환점      |
| lg     | 1024px 이상     | 노트북/PC 화면 대응            

- **기본**: 모바일 기준 (prefix 없음)  
- **sm**: 작은 태블릿 이상  
- **md**: 태블릿 or 노트북 이상  
- **lg**: 데스크탑 이상  

---

## ⚙️ 언제 어떤 prefix를 쓰는지 예시

| 상황                            | 추천 prefix     |
| ------------------------------- | --------------- |
| 모바일에선 세로, 태블릿부턴 가로로 정렬 | `sm:flex-row`    |
| 모바일에선 한 줄, PC부턴 2열 grid        | `md:grid-cols-2` |
| 모바일에선 글자 작게, PC에선 크게        | `lg:text-lg`    |
| 모바일에서 숨기고 PC에서만 보이기        | `hidden sm:block` |

---

## ✅ 요약 정리

Tailwind로 반응형을 구성할 땐 이렇게 생각하세요:

> **모바일 먼저 만들고, 필요한 지점에서만 sm, md, lg로 점진적으로 확장한다.**

실제로는 아래 3단계만 잘 활용해도 웬만한 반응형 웹앱 대부분 커버할 수 있습니다:

```txt
기본 스타일 (모바일)
→ sm: 태블릿
→ md: 노트북
→ lg: 데스크탑
```
