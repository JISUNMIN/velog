## 🧩 VPN의 정의

**VPN (Virtual Private Network, 가상 사설망)**은  
공용 인터넷망 위에 ‘가상의 전용 통로(Tunnel)’를 만들어  
외부에서도 내부 네트워크처럼 안전하게 접속할 수 있게 하는 기술입니다.

쉽게 말하면, **멀리서도 회사 내부망에 들어가는 ‘보안 터널’**을 만들어주는 것입니다.

---

## 1. VPN은 가상 사설망(Virtual Private Network)입니다

**VPN**은 영어로 **Virtual Private Network**,  
한글로는 **가상 사설망**이라고 부릅니다.

말 그대로 **“가상의 전용 네트워크를 만드는 기술”**입니다.  
즉, 실제 물리적인 전용선 없이도  
인터넷 위에 **보안 터널**을 만들어 내부 네트워크처럼 연결해 줍니다.

> 💡 예: 집에서도 회사 내부망처럼 접속 가능 → “가상(virtual)” + “사설망(private network)”

---

## 2. VPN은 인프라의 한 부분입니다

**인프라(Infrastructure)**는 IT에서  
서비스가 동작하기 위한 기반 시스템 전체를 의미합니다.

예를 들어 다음과 같이 나뉩니다:

- **네트워크 인프라** → 라우터, 방화벽, **VPN**, 프록시 서버 등  
- **서버 인프라** → 애플리케이션 서버, 데이터베이스(DB) 서버, 스토리지 등  
- **클라우드 인프라** → AWS, GCP, Azure 등에서 제공하는 네트워크·보안 자원

이 중에서 **VPN은 네트워크 인프라의 구성 요소 중 하나**입니다.  
즉, 인프라 담당자(보안/네트워크 엔지니어)가 관리하는 **보안 통신 인프라**에 해당합니다.



| 항목 | 설명 |
|------|------|
| **VPN 뜻** | Virtual Private Network (가상 사설망) |
| **역할** | 외부와 내부망을 안전하게 연결하는 보안 터널 |
| **소속 인프라 영역** | 네트워크 인프라(Network Infrastructure) |
| **관리 주체** | 인프라 / 보안 / 네트워크 엔지니어 |
| **핵심 목적** | 내부 자원 보호 및 안전한 통신 제공 |



> 💬 **VPN은 네트워크 인프라의 한 구성 요소이자,  
> 인터넷 위에 만들어진 ‘가상 사설망(Virtual Private Network)’입니다.**

---


## 💡 VPN의 기본 원리

VPN은 사용자의 PC와 회사 내부망 사이에 **암호화된 연결 통로**를 형성합니다.
[내 PC] ⇄ 🌐 [VPN 서버] ⇄ [회사 내부망]



이 연결을 통해 회사 외부(집, 카페, 출장지 등)에서도  
사내 시스템(GitLab, DB, 내부 API 등)에 접근할 수 있습니다.

---

## 🌐 VPN, IP, 그리고 내부망의 관계

VPN을 이해할 때 가장 자주 헷갈리는 부분이 바로  
“VPN이 **내 IP를 회사 내부망 IP로 바꿔주는 건가?**” 하는 부분입니다.

정확히는 다음과 같습니다 👇

- VPN은 **내 IP 자체를 172.x.x.x 같은 내부 주소로 바꾸는 것이 아닙니다.**  
  대신, **그 내부 IP들로 ‘갈 수 있는 길(경로, Route)’을 추가**해주는 기술입니다.  
- VPN을 켜면 내 컴퓨터는 `192.168.255.10` 같은 **가상 내부 IP**를 부여받고,  
  회사 내부망(예: `172.20.x.x`, `172.32.x.x`)으로 트래픽을 보낼 수 있게 됩니다.  
- 이렇게 만들어진 통로를 통해 회사의 서버, DB, Git 등 내부 시스템에 접근합니다.


> 💡 **VPN은 내 주소를 바꾸는 게 아니라,  
> 특정 네트워크(회사 내부망)로 가는 안전한 길을 열어주는 기술**입니다.

---

## 🧭 회사에서 VPN을 사용하는 이유

### 1️⃣ 외부 접근 차단을 위한 보안 강화
회사의 내부 시스템(Git, DB, 개발 서버 등)은  
외부 인터넷에서는 접속할 수 없도록 막혀 있습니다.  
대신 VPN을 통해서만 접근할 수 있게 하여 **보안 사고를 예방합니다.**


### 2️⃣ 회사 내부에서도 VPN을 쓰는 이유

“회사 안인데 왜 VPN을 또 켜야 하지?”라는 의문이 들 수 있습니다.  
하지만 회사 내부망은 하나의 통합망이 아니라, 보안을 위해 **여러 구역으로 나뉘어 있습니다.**

| 구역 | 설명 |
|------|------|
| **업무망** | 일반 사무용 네트워크 (메일, 메신저 등) |
| **개발망** | 개발 서버, 데이터베이스 등 민감한 시스템 |
| **운영망** | 실제 서비스 운영 서버 및 배포 환경 |

업무망에 있는 PC는 기본적으로 개발망이나 운영망에 직접 접근할 수 없습니다.  
보안을 위해 **망 사이에 벽(방화벽)**이 있기 때문입니다.

이때 VPN을 켜면  
👉 업무망에 있더라도 **보안 인증을 거쳐 개발망 또는 운영망으로 들어갈 수 있게** 됩니다.

즉,    **회사 내부에서도 VPN은 ‘보안 구역 간 이동을 위한 열쇠’ 역할을 할 수 있습니다.**


### 3️⃣ 접속 기록과 사용자 통제
VPN을 사용하면 누가, 언제, 어떤 서버에 접속했는지 모두 기록됩니다.  
이를 통해 보안 사고를 추적하고, 필요할 경우 특정 사용자의 접근을 차단할 수 있습니다.

### 4️⃣ 내부 통신 암호화
같은 회사 내부망이라도 완전히 안전한 것은 아닙니다.  
VPN을 사용하면 내부에서도 데이터가 암호화되어,  
**패킷 감청(스니핑)이나 정보 유출 위험을 줄일 수 있습니다.**

## 📦 [NPM Private Registry](https://velog.io/@sunmins/Private-Registry%EB%9E%80-%EC%98%A4%ED%94%88%EC%86%8C%EC%8A%A4%EC%99%80-%EB%82%B4%EB%B6%80-%EB%9D%BC%EC%9D%B4%EB%B8%8C%EB%9F%AC%EB%A6%AC%EB%A5%BC-%ED%95%A8%EA%BB%98-%EB%8B%A4%EB%A3%A8%EB%8A%94-%EC%82%AC%EB%82%B4-%EC%A0%80%EC%9E%A5%EC%86%8C-%EC%9D%B4%ED%95%B4%ED%95%98%EA%B8%B0)란?

> “패키지를 회사 내부 전용으로 관리하는 저장소”를 말합니다.

회사에서는 보안상 이유로  
외부에서 직접 npm 공식 서버(`https://registry.npmjs.org`)에 접속하지 않고,  
**내부망 안에 전용 npm 저장소(Private Registry)**를 운영하기도 합니다.  

예를 들어,  
사내 네트워크 전용 주소인 `http://172.20.10.5:4873` 같은 형태입니다.

이런 **사내 Private Registry**는 다음과 같은 이유로 사용됩니다.

- 회사에서 만든 **내부 전용 라이브러리**를 외부에 공개하지 않기 위해  
- 외부 인터넷이 제한된 환경에서도 **내부 캐시를 통해 빠르게 설치하기 위해**  
- 패키지 버전과 접근 권한을 **중앙에서 통제하기 위해**

> 💡 즉, 내부 개발자만 접근할 수 있는 전용 npm 서버를 두고,  
> 이 서버에 접근하려면 **VPN을 통해 내부망에 연결해야** 합니다.  
> VPN은 이러한 **내부 전용 시스템에 접근하기 위한 보안 통로** 역할을 합니다.



| 항목 | 설명 |
|------|------|
| **VPN 뜻** | Virtual Private Network (가상 사설망) |
| **외부에서의 역할** | 외부에서도 내부망처럼 접속 가능하게 함 |
| **내부에서의 역할** | 망 분리 환경에서 다른 보안 구역으로 이동할 때 사용 |
| **Private Registry** | 회사 내부에서만 접근 가능한 npm 패키지 저장소 |
| **VPN이 필요한 이유** | 내부 시스템 보호, 접근 제어, 통신 암호화, 규제 준수 |

## 🗝️ 한 줄 요약

> 💬 **VPN은 ‘보안 구역으로 들어가는 통제된 터널’입니다.**  
> 외부에서는 내부망으로, 내부에서는 구역 간 이동을 위해 사용했습니다.  
> 사내 레지스트리처럼 내부 전용 시스템에 접근할 때도 VPN이 필요했습니다.
