<p>서적을 읽던 중 GitHub Pages로 만든 포트폴리오 페이지가 너무 눈에 띄어 저도 한 번 만들어 보기로 했습니다.</p>
<p>GitHub Pages는 <strong>정적 사이트를 무료로 배포</strong>해 주는 서비스로,<br />개인 포트폴리오를 보여주기 좋은 환경을 제공합니다.</p>
<p>그래서 GitHub Pages를 이용해<br />포트폴리오 페이지를 직접 만들어 보기로 했습니다.<br />이번글에서는 먼저 기본 설정부터 진행해 보겠습니다. 
렛츠고 ~</p>

<h2 id="1-github-repository-생성">1. GitHub Repository 생성</h2>
<p>먼저 GitHub에서 새로운 Repository를 생성했습니다.</p>
<p><img alt="repository 생성" src="https://velog.velcdn.com/images/sunmins/post/11df7fcc-7c01-407f-aa55-2fd3e9678231/image.png" /></p>
<p><img alt="repository 이름 설정" src="https://velog.velcdn.com/images/sunmins/post/cbd0aa08-9b18-4fe8-9f53-30116bb85d2f/image.png" /></p>
<p>Repository 이름은 반드시 아래 형식으로 설정해야 합니다.</p>
<blockquote>
<p>본인아이디.github.io</p>
</blockquote>
<p>이 형식으로 생성해야 GitHub Pages가 <strong>자동으로 활성화</strong>됩니다.</p>

<h2 id="2-indexhtml-파일-생성">2. index.html 파일 생성</h2>

<p><img alt="new file 클릭" src="https://velog.velcdn.com/images/sunmins/post/b1440068-9c1c-424c-881c-b32565f209e3/image.png" /></p>
<p>index.html을 생성합니다.</p>
<blockquote>
<p><code>index.html</code>은 GitHub Pages의 <strong>기본 진입 파일</strong>입니다.</p>
</blockquote>
<p><img alt="index.html 생성" src="https://velog.velcdn.com/images/sunmins/post/f0c85089-07d7-4a99-92ae-3adb30826bcd/image.png" /></p>

<h2 id="3-github-pages-배포-확인">3. GitHub Pages 배포 확인</h2>
<p>파일 생성 후,<br />해당 Repository의 <strong>Settings → Pages</strong> 메뉴로 이동했습니다.
Pages 설정 화면에서 배포된 주소를 확인했습니다.</p>
<p><img alt="pages 설정" src="https://velog.velcdn.com/images/sunmins/post/46903007-dfaf-40e3-9bb5-1393aed18708/image.png" /></p>
<p><img alt="배포 주소 확인" src="https://velog.velcdn.com/images/sunmins/post/1477b9c8-ca88-45aa-8c15-3742d4b8b8ae/image.png" /></p>
<p>정상적으로 배포된 것을 확인할 수 있었습니다.</p>

<h2 id="4-html-작성-환경-선택">4. HTML 작성 환경 선택</h2>
<p>기본으로 생성된 mock layout을 제거한 뒤,<br />HTML 코드를 작성하기 위해 online code editor를 선택 합니다.</p>
<p>online code editor를 사용하면<br />별도의 환경 설정 없이 코드를 바로 작성할 수 있고,<br />작성한 내용을 화면으로 확인할 수 있어 편리하게 개발 할 수 있습니다.</p>
