## Windows에서 Oh My Posh로 예쁜 터미널 만들기

기본 VS Code 터미널, 너무 밋밋하지 않나요? 게다가 현재 Git 브랜치조차 표시되지 않아서 실수하기 딱 좋습니다.
`Oh My Posh`를 사용하면 현재 브랜치, 경로, 권한 상태 등을 아이콘과 컬러로 한눈에 확인할 수 있어 예쁘기만 한 게 아니라, 생산성도 올라갑니다.

> Mac이나 Linux는 설치 방법이 다르니 [공식 홈페이지](https://ohmyposh.dev/docs/installation)에서 별도 확인해 주세요.

---

## 1. Oh My Posh 설치하기

- [Oh My Posh 공식 Windows 설치 페이지](https://ohmyposh.dev/docs/installation/windows)에 접속해 설치 파일을 받으실 수 있습니다.  
- 저는 Microsoft Store에서 설치했습니다. 명령어로도 설치 가능하지만, Store가 가장 간편합니다.  
  ![](https://velog.velcdn.com/images/sunmins/post/dc596f1d-638d-4f57-8896-4176c0e64300/image.png)

## 2. 테마 확인 및 임시 적용

- 사용 가능한 테마 목록은 [공식 테마 페이지](https://ohmyposh.dev/docs/themes)에서 확인할 수 있습니다.  
- 저는 `hul10` 테마를 사용했습니다.

### 임시로 테마 적용하기 (PowerShell 기준)

```powershell
oh-my-posh init pwsh --config "C:\Users\sunmi\custom_themes\hul10.omp.json" | Invoke-Expression
```
마음에 들면 프로필에 적용해 터미널 실행 시 자동으로 적용되도록 설정할 수 있습니다.

## 3. PowerShell 프로필에 테마 적용하기

터미널에서 아래 명령어를 입력해 프로필 파일을 엽니다.

```powershell
notepad $PROFILE
```
메모장이 열리면 아래 내용을 추가하고 저장합니다.
```powershell
oh-my-posh init pwsh --config "$env:POSH_THEMES_PATH/hul10.omp.json" | Invoke-Expression

```
### 프로필 파일이 없다는 오류가 발생한다면?
아래 명령어로 프로필 파일을 생성 후 다시 열어주세요.
```powershell
New-Item -ItemType File -Path $PROFILE -Force
notepad $PROFILE
```
## 4. Nerd Font (필수!) 설치하기
Oh My Posh에서 사용하는 아이콘(,  등)은 일반 폰트에서는 깨져 보입니다.
따라서 Nerd Font 설치는 필수입니다.


### 설치 방법

터미널에 아래 명령어를 입력하면 설치 가능한 Nerd Font 목록이 나타납니다:

```powershell
oh-my-posh font install
```

목록에서 원하는 폰트를 선택하여 설치할 수 있습니다.
가장 많이 사용하는 Meslo 폰트를 추천합니다.

직접 설치하려면 아래와 같이 입력하세요.

```powershell
oh-my-posh font install meslo
```

 제어판 > 글꼴에서 설치된 폰트를 직접 확인하실 수 있습니다.


## 5.VS Code 터미널 폰트 설정
VS Code 터미널에서도 Nerd Font를 지정해야 아이콘이 깨지지 않고 제대로 보입니다.

VS Code 설정(settings.json)에 다음을 추가해 주세요:

단축키: ctrl + shift+ P
```json
"terminal.integrated.fontFamily": "MesloLGM Nerd Font"
```
주의: 폰트 이름은 정확히 설치한 Nerd Font 이름과 일치해야 합니다.

## ❓ PowerShell 스크립트 실행 정책 오류시 해결 방법
터미널 실행 시 아래와 같은 에러가 발생할 수 있습니다: 이 시스템에서 스크립트를 실행할 수 없으므로 ... 파일을 로드할 수 없습니다.

이는 PowerShell 스크립트 실행 정책이 제한되어 있기 때문입니다.

해결 방법:
PowerShell을 관리자 권한으로 실행합니다.
→ 시작 메뉴 → PowerShell → 우클릭 → '관리자 권한으로 실행'

아래 명령어 입력:

```powershell
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
```
RemoteSigned는 로컬에서 작성한 스크립트는 실행 가능하고,
인터넷에서 다운로드한 스크립트는 서명이 있어야 실행됩니다.
대부분 사용자에게 안전하면서도 실용적인 설정입니다.

변경 승인 시 Y를 눌러 승인합니다.
