<li>전원 ON</li>

<li>사용자 프로그램 실행</li>

<h3 id="os는-왜-저장장치에-있을까">OS는 왜 저장장치에 있을까?</h3>
<p>운영체제(OS)는 <strong>전원이 꺼져도 사라지면 안 되는 핵심 소프트웨어</strong>이다.<br />RAM은 휘발성 메모리이기 때문에 전원이 꺼지면 내용이 모두 사라진다.</p>
<p>따라서 OS는 반드시 <strong>비휘발성 저장장치(SSD/HDD)</strong> 에 저장되어 있어야 하며,<br />부팅 시 저장장치에 있는 OS를 <strong>RAM으로 적재</strong>한 뒤 실행된다.</p>
</blockquote>

┌──────────────────────────┐

│  ┌────────────────────┐ │
│  │     레지스터      │  │  ← CPU 내부
│  └────────────────────┘  │
│  ┌────────────────────┐  │
│  │  캐시 (L1/L2/L3)   │ │  ← CPU 내부

└───────────┬──────────────┘
            ↓
┌──────────────┐

       ↓   OS 위치 탐색

<h2 id="메모리-계층-cpu-기준">메모리 계층 (CPU 기준)</h2>
<p>CPU는 모든 데이터를 한 번에 접근하지 않고,<br /><strong>속도와 용량에 따라 계층적인 메모리 구조</strong>를 사용한다.</p>
<pre><code class="language-text">(가장 빠름, 용량적음, 가장 비쌈)
┌──────────────┐
│  레지스터    │  ← CPU 내부
└──────────────┘
┌──────────────┐
│   캐시       │  ← L1 / L2 / L3 (CPU 내부)
└──────────────┘
┌──────────────┐
│    RAM       │  ← 주기억장치
└──────────────┘
┌──────────────┐
│ SSD / HDD    │  ← 저장장치(비휘발성)
└──────────────┘
(가장 느림, 용량 큼, 가장 저렴)</code></pre>
<pre><code class="language-text">CPU가 어떤 값을 필요로 함
↓
레지스터에 있나? → YES → 사용
↓ NO
캐시에 있나?     → YES → 레지스터로 복사 → 사용 
↓ NO
RAM에 있나?      → YES → 캐시로 복사 → 레지스터로 복사 → 사용
↓ NO
저장장치 접근     → YES → OS가 해당 데이터를 RAM에 적재 → 캐시로 복사 → 레지스터로 복사 → 사용</code></pre>
<p>메모리 계층은 자주 사용하는 데이터를 가장 가까운 곳에 두고,
없을 경우에만 점점 더 먼 저장소로 내려가며 찾는 구조이다.</p>
<h2 id="cpu-포함-구성요소">CPU 포함 구성요소</h2>
<ul>
<li>레지스터 : CPU 내부에서 즉시 연산에 사용되는 저장공간</li>
<li>캐시(L1/L2/L3): CPU와 RAM 사이의 고속 메모리 </li>
</ul>
<h2 id="cpu-외부-메모리">CPU 외부 메모리</h2>
<ul>
<li>RAM (주기억장치): 실행 중인 OS와 프로그램이 올라가는 주기억장치</li>
<li>SSD/HDD (저장장치): OS와 프로그램을 영구 저장하는 저장장치</li>
</ul>
<hr />
<h2 id="✅-정리">✅ 정리</h2>
<ul>
<li><strong>CPU</strong>는 레지스터와 캐시를 포함한 <strong>실행 장치</strong>이다.</li>
</ul>
<p>컴퓨터 전원이 켜지면 다음과 같은 순서로 동작한다.</p>
<ol>
<li><strong>CPU가 동작을 시작</strong>한다.</li>
<li>CPU는 <strong>ROM에 있는 BIOS(UEFI) 부팅 코드</strong>를 실행한다.</li>
<li>BIOS는 <strong>저장장치에 있는 OS를 찾아 RAM에 적재</strong>한다.</li>
<li><strong>CPU는 RAM에 올라온 OS를 실행</strong>한다.</li>
<li><strong>OS는 CPU를 사용해 사용자 프로그램을 실행</strong>한다.</li>
</ol>
