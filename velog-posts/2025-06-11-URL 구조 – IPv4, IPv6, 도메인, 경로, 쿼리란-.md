URL, 프로토콜, 도메인, 포트, IP주소, IPv4, IPv6, 경로, 쿼리등에 대해 알아보자.


## 📎 URL이란?

**URL (Uniform Resource Locator)** 은 웹에서 리소스의 위치를 나타내는 주소

✅ URL 구성 요소
예시 URL:
```
https://www.example.com:3000/api/user?id=123#section1
```

📌 각 부분 설명

| 구성 요소            | 예시 부분             | 설명                                                    |
| ---------------- | ----------------- | ----------------------------------------------------- |
| **프로토콜**         | `https://`        | 컴퓨터나 네트워크 장치들이 서로 통신할 때(데이터) 약속하는 규칙으로 데이터가 어떻게 주고 받아질지에 대한 언어같은 것 (`http`, `https`, `ftp` 등)       |
| **도메인**          | `www.example.com` | 요청을 보낼 서버의 주소로 IP 대신 사람이 읽기 좋은 형태임 ex) 도메인 = 집의 이름표(예:홍길동 집)   
| **IP 주소**          | `192.168.1.1(IPv4), 2001.0db8::1(IPv6)` | 네트워크에서 컴퓨터나 서버를 식별하는 숫자 주소 예) IP주소 = 집의 실제 주소(서울시 강남구...) |
| **포트**           | `:3000`           | (생략 가능) 서버가 사용하는 포트 번호 웹 기본은 80(HTTP), 443(HTTPS) |
| **경로(Route)**    | `/api/user`       | 서버 내에서 어떤 리소스를 요청하는지를 나타냄                          |
| **쿼리(Query)**    | `?id=123`         | 추가적인 요청 정보를 key-value 형태로 전달                      |
| **해시(Fragment)** | `#section1`       | 페이지 내 특정 위치로 이동 (브라우저 전용, 서버에는 전달되지 않음)               |


🌐 IP 주소 vs 도메인 (URL에 동시에 들어가지 않음)

| 구분         | 설명                                                     | 예시                                          | 사용
| ---------- | ------------------------------------------------------ | ------------------------------------------- |----------------------------|
| **IP 주소**  | 네트워크에서 컴퓨터나 서버를 식별하는 숫자 주소. 인터넷상의 '집 주소' 같은 역할.        | `http://192.168.0.1:3000 => 192.168.1.1` (IPv4), `http://[2001:0db8::1]:8080/path/to/resource?query=abc#footer` => `2001:0db8::1` (IPv6) | 주로 내부 네트워크 접속이나 서버 직접 접속힐때 사용
| **도메인 이름** | IP 주소 대신 사람이 기억하기 쉽게 만든 문자 주소. DNS가 도메인을 IP 주소로 변환해준다. | `www.example.com`                           | 사람이 기억하기 쉬운 형태로 접속하는 일반적인 경우


📡 흔히 쓰이는 프로토콜 종류

| 프로토콜    | 설명                          |
| ------- | --------------------------- |
| `http`  | 웹 페이지를 불러올 때 쓰는 프로토콜 (비암호화) |
| `https` | 암호화된 HTTP, 보안이 강화된 프로토콜     |
| `ftp`   | 파일 전송용 프로토콜                 |
| `smtp`  | 이메일 보내기용 프로토콜               |

🔢 포트 번호에 대해

기본 포트 80,443은 운영체제에서 관리자 권한이 필요해서 일반 개발자가 바로 열기 어려움, 그래서 임시 포트를 띄우고 개발이나 테스트를 진행

| 포트 번호 | 용도                 | 설명                  |
| ----- | ------------------ | ------------------- |
| 80    | HTTP 기본 포트         | 비암호화 웹 트래픽의 표준 포트   |
| 443   | HTTPS 기본 포트        | 암호화된 웹 트래픽 표준 포트    |
| 3000  | 개발용 포트 (Node.js 등) | 주로 로컬 개발 환경에서 많이 사용, React,Node.js,Next.js 등... |
| 8080  | 개발용 혹은 대체 웹서버 포트   | 기본 80 포트를 대신해서 쓰임, 톰캣같은 자바 웹서버   |
