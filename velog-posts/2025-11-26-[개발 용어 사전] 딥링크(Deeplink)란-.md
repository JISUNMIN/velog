<p><strong>딥링크(Deeplink)</strong>는 <strong>앱의 특정 화면으로 바로 이동할 수 있게 해주는 주소(URL, 커스텀 스킴)</strong>입니다.<br />일반적인 웹 링크가 웹페이지를 여는 것처럼,
딥링크는 <strong>해당 앱의 특정 기능이나 화면을 곧바로 열어주는 링크</strong>입니다.</p>
<blockquote>
<p>“앱의 특정 위치를 바로 여는 전용 주소”</p>
</blockquote>
<p>라고 이해할 수 있습니다.</p>
<hr />
<h2 id="💡-왜-딥링크를-사용할까">💡 왜 딥링크를 사용할까?</h2>
<h3 id="1-앱의-특정-화면으로-빠르게-이동">1) 앱의 특정 화면으로 빠르게 이동</h3>
<p>일반적으로 앱을 켜면 홈 화면 → 메뉴 → 상세 화면…<br />이런 흐름을 따라가야 합니다.</p>
<p>하지만 딥링크를 사용하면:</p>
<ul>
<li>바로 상품 상세 페이지  </li>
<li>특정 이벤트 화면  </li>
<li>설정 화면  </li>
<li>특정 유저 프로필</li>
</ul>
<p>등으로 <strong>즉시 이동</strong>할 수 있습니다.</p>
<p>→ <strong>사용자 경험(UX) 향상 + 이동 시간 단축</strong></p>
<hr />
<h3 id="2-다양한-진입-경로에서-앱으로-이동-가능">2) 다양한 진입 경로에서 앱으로 이동 가능</h3>
<p>딥링크는 “앱에서만 쓰는 기술”이 아니라,<br /><strong>앱을 여는 데 사용되는 링크 기술</strong>입니다.<br />즉, 딥링크는 <strong>출발 지점이 무엇이냐가 아니라 ‘도착지가 앱의 특정 화면이냐’가 핵심</strong>입니다.</p>
<p>가능한 이동 형태는 다음과 같습니다:</p>
<ul>
<li><strong>웹 → 앱 이동</strong>  </li>
<li><strong>앱 → 앱 이동</strong>  </li>
<li><strong>메시지/SMS → 앱 이동</strong>  </li>
<li><strong>알림(푸시) → 앱 이동</strong></li>
</ul>
<p>즉,  </p>
<blockquote>
<p><strong>딥링크는 ‘다양한 외부 경로에서 앱의 특정 위치로 진입시키는 기술’이다.</strong></p>
</blockquote>
<p>반대로 아래 이동은 딥링크가 아닙니다:</p>
<ul>
<li>웹 → 웹 이동 (웹 링크)  </li>
<li>앱 → 웹 이동 (외부 브라우저 열기 또는 WebView)  </li>
</ul>
<p>딥링크는 <strong>항상 앱의 특정 화면을 여는 목적</strong>으로 사용됩니다.</p>
<hr />
<h3 id="3-마케팅·알림·공유-기능에-필수">3) 마케팅·알림·공유 기능에 필수</h3>
<p>딥링크는 다음과 같은 기능에 반드시 필요합니다:</p>
<ul>
<li>푸시 알림 클릭 시 특정 화면 열기  </li>
<li>카카오톡 공유에서 앱의 특정 화면으로 이동  </li>
<li>카카오/네이버 로그인 후 앱으로 돌아오기  </li>
<li>광고 배너 → 앱 내 특정 화면  </li>
<li>웹에 “앱으로 보기” 버튼 만들기</li>
</ul>
<p>딥링크 없이는 이런 기능을 구현할 수 없습니다.</p>
<hr />
<h2 id="📱-딥링크-예시">📱 딥링크 예시</h2>
<h3 id="✔️-카카오톡-앱에서-쿠폰-받기-링크-클릭">✔️ 카카오톡 앱에서 “쿠폰 받기” 링크 클릭</h3>
<p>쿠폰을 누르면 <strong>쿠팡 앱이 자동으로 실행되고</strong>,  
바로 해당 쿠폰 이벤트 페이지로 이동하는 경우 → <strong>딥링크</strong></p>
<h3 id="✔️-배민-알림-클릭">✔️ 배민 알림 클릭</h3>
<p>“오늘의 특가 보기” 알림을 누르면<br />앱이 열리고 바로 특가 페이지로 이동 → <strong>딥링크</strong></p>
<h3 id="✔️-카카오톡-채팅방에서-앱-초대-링크">✔️ 카카오톡 채팅방에서 앱 초대 링크</h3>
<p><code>myapp://invite/123</code> 같은 주소를 눌러 앱의 특정 초대 화면으로 이동 → <strong>딥링크</strong></p>
<hr />
<h2 id="🔍-딥링크와-라우터의-차이">🔍 딥링크와 라우터의 차이</h2>
<p>둘은 쉽게 혼동되지만 역할이 다릅니다.</p>
<h3 id="✔️-라우터router">✔️ 라우터(router)</h3>
<ul>
<li><strong>앱 내부에서 화면 간 이동을 관리</strong></li>
<li>React Navigation, Flutter Router, Android Navigation 등</li>
<li>앱이 이미 켜져 있다는 전제</li>
</ul>
<p>예: <code>/home → /product/123 → /cart</code></p>
<h3 id="✔️-딥링크deeplink">✔️ 딥링크(deeplink)</h3>
<ul>
<li><strong>앱 외부에서 앱 내부 특정 화면으로 진입하는 주소</strong></li>
<li>앱이 꺼져 있어도, 설치되어 있어도/없어도 동작</li>
</ul>
<p>예:  </p>
<ul>
<li>웹 링크 클릭 → 앱 특정 화면으로 열림  </li>
<li>카톡 링크 클릭 → 앱 특정 화면으로 이동  </li>
<li>다른 앱에서 현재 앱으로 이동</li>
</ul>
<p>즉,</p>
<blockquote>
<p><strong>라우터 = 내부 이동</strong><br /><strong>딥링크 = 외부 → 내부 진입</strong></p>
</blockquote>
<hr />
<h2 id="📎-딥링크의-종류-간단-정리">📎 딥링크의 종류 간단 정리</h2>
<h3 id="✔️-커스텀-스킴custom-scheme">✔️ 커스텀 스킴(Custom Scheme)</h3>
<p>예: <code>myapp://product/123</code></p>
<ul>
<li>앱에서 정의한 전용 프로토콜(스킴)을 사용하는 방식  </li>
<li>구현이 간단하고 앱 내부에서 처리하기 편함  </li>
<li><strong>앱이 설치되어 있어야만 동작</strong> (설치 안 되어 있으면 열리지 않음)  </li>
<li>브라우저나 일부 플랫폼에서는 인식이 제한될 수 있음  </li>
<li>주로 앱 내부 이동 또는 앱 간 연동에서 사용</li>
</ul>
<hr />
<h3 id="✔️-유니버설-링크ios--앱-링크android">✔️ 유니버설 링크(iOS) / 앱 링크(Android)</h3>
<p>예: <code>https://myapp.com/product/123</code></p>
<ul>
<li><strong>일반 웹 URL처럼 생겼지만 앱과 자동 연결되는 방식</strong>  </li>
<li>앱이 설치되어 있으면 → 해당 앱의 특정 화면이 열리고  </li>
<li>앱이 설치되지 않았으면 → 정상적으로 웹 페이지가 열림  </li>
<li>앱/웹 양쪽 상황을 자연스럽게 처리하기 때문에 가장 많이 쓰임  </li>
<li>마케팅 링크, 메시지 공유, 푸시알림 등에 최적화  </li>
<li>웹·앱 둘 다 같은 URL 체계를 사용할 수 있어 유지보수도 쉬움</li>
</ul>
<blockquote>
<p>딥링크는 <strong>앱이 설치되어 있어야만 동작하는 것</strong>도 있고,
<strong>앱이 없어도 웹으로 fallback 되는 것</strong>도 있다.
가장 현대적인 방식은 <strong>앱 설치 여부와 상관없이 동작하도록 만든
유니버설 링크 / 앱 링크 방식입니다.</strong></p>
</blockquote>
<hr />

<blockquote>
<p>딥링크는 앱의 특정 화면을 바로 여는 전용 주소이며,<br />웹·앱 어디에서든 <strong>앱 내부로 즉시 진입</strong>할 수 있게 해주는 핵심 기술이다.</p>
</blockquote>
