<h2 id="문제-상황">문제 상황</h2>
<p>오랜만에 접속한 <strong>Next.js 프로젝트</strong>에서 데이터가 정상적으로 표시되지 않는 현상을 발견했습니다.<br />브라우저 <strong>개발자 콘솔</strong>을 확인해보니 API 요청 시 <strong>500 에러</strong>가 발생하고 있었습니다.
이전까지 문제없이 동작하던 프로젝트였기에, 무엇이 달라졌는지 확인해볼 필요가 있었습니다.<img alt="" src="https://velog.velcdn.com/images/sunmins/post/a0de5edd-c3f3-4462-ae11-29e803873e80/image.png" /></p>
<hr />
<h2 id="확인-사항">확인 사항</h2>
<ul>
<li>Next.js 프로젝트는 정상적으로 <strong>배포 완료</strong></li>
<li>배포 플랫폼에서 <strong>Ready 상태</strong> 확인</li>
<li>서버 자체 로그에서는 특별한 오류가 보이지 않음</li>
</ul>
<p>위 상황으로 보아 Next.js 자체 문제보다는 <strong>외부 서비스 문제</strong>일 가능성이 높다고 판단했습니다.</p>
<h2 id="원인-분석">원인 분석</h2>
<p>프로젝트에서 사용 중인 <strong>Supabase</strong>가 원인일 가능성을 의심했습니다.</p>
<ul>
<li>Supabase <strong>무료 플랜</strong> 사용 중</li>
<li>최근 약 <strong>일주일 이상 프로젝트 미접속</strong></li>
<li>Supabase에서 “일정 기간 미사용 시 프로젝트가 Paused 된다”는 안내 메일 수신을 계속 받았었음</li>
</ul>
<p>Supabase 콘솔에 접속해 확인한 결과,<br />프로젝트 상태가 <strong>Paused</strong> 상태로 전환되어 있음을 확인했습니다.</p>
<h2 id="원인">원인</h2>
<ul>
<li>Supabase 무료 플랜 정책으로 인해<br /><strong>일정 기간 미사용 시 프로젝트가 자동으로 Paused 상태로 변경</strong></li>
<li>Paused 상태에서는 API 요청이 정상적으로 처리되지 않아<br />Next.js에서 <strong>500 에러</strong>가 발생</li>
</ul>
<h2 id="✅-해결-방법">✅ 해결 방법</h2>
<ol>
<li>Supabase 콘솔 접속</li>
<li>Paused 상태인 프로젝트 확인</li>
<li><strong>Resume Project</strong> 버튼 클릭하여 프로젝트 재시작</li>
<li>일정 시간 지난 후 Supabase Dashboard 복구됨</li>
<li>Next.js 프로젝트에서 데이터 정상 출력 확인</li>
</ol>
