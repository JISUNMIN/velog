## ✅ 웹(Web)이란?

**웹(Web, World Wide Web, WWW)**은  
인터넷에 연결된 컴퓨터들이 서로 정보를 공유하고 찾아볼 수 있는 **공간**을 의미합니다.

전 세계에 흩어진 정보가 마치 **거미줄처럼 연결**되어 있어,  
웹을 통해 원하는 정보를 빠르게 찾을 수 있습니다.

---

## 🔗 하이퍼텍스트(HyperText)란?

웹에서는 다양한 정보를 주고받기 위해 **일련의 규칙**을 따릅니다.  
그중 하나는 웹 문서들이 **하이퍼텍스트 형식으로 서로 연결**된다는 점입니다.

- **하이퍼텍스트**:  
  다른 문서로 이동할 수 있는 **링크(하이퍼링크)**를 포함한 문서입니다.

- **하이퍼링크(Hyperlink)**:  
  한 문서에서 다른 문서로 넘어갈 수 있도록 해주는 **연결 고리**입니다.

- **웹페이지(Web Page)**:  
  하이퍼텍스트 형식으로 작성된 **하나의 웹 문서**입니다.

- **웹서핑(Web Surfing)**:  
  하이퍼링크를 따라 여러 웹페이지를 **이동**하는 행위를 의미합니다.

---

## 🌍 인터넷과 웹의 차이

| 항목 | 인터넷 | 웹(Web) |
|------|--------|---------|
| 개념 | 컴퓨터와 컴퓨터가 연결된 **네트워크 인프라** | 인터넷 위에서 동작하는 **정보 공유 시스템** |
| 비유 | **고속도로** | 고속도로 위를 달리는 **자동차** |
| 역할 | 데이터를 전달하는 **통신망** | 정보 검색, 콘텐츠 공유 등을 **가능하게 함** |

- **인터넷**은 인프라 (정보의 바다)이고,  
- **웹**은 그 위에서 정보를 공유할 수 있도록 만든 시스템입니다.

---

## 🧱 웹 애플리케이션 아키텍처 (Web Application Architecture)

웹이 작동하기 위해 필요한 요소들과 이들의 **상호작용 구조**를 의미합니다.

### 🔸 웹 클라이언트 (Web Client)

> 레스토랑에서 **주문을 요청하는 손님**

- 사용자와 직접 상호작용합니다.  
- 브라우저를 통해 서버에 요청을 보냅니다.

### 🔸 웹 서버 (Web Server)

> 레스토랑의 **주방장**

- 정적인 콘텐츠(HTML, CSS, 이미지 등)를 클라이언트에 전달합니다.  
- 요청을 받아 필요한 파일 또는 처리를 반환합니다.

### 🔸 WAS (Web Application Server)

> 주방장의 지시에 따라 일하는 **보조 요리사**

- 동적인 콘텐츠 처리를 담당합니다 (예: 로그인, DB 연결 등).  
- 웹 서버로부터 요청을 받아 애플리케이션 로직을 처리한 뒤 결과를 반환합니다.

## 📈 웹의 발전 과정 (Web 1.0 → Web 3.0)

### 🕸 Web 1.0  
> 대표: 야후, 네이버, 다음

- 읽기 전용 (Read-only) 웹  
- 콘텐츠 소비 중심  
- 사용자 상호작용 거의 없음

---

### 🔄 Web 2.0  
> 대표: 페이스북, 유튜브, 인스타그램

- 읽기 + 쓰기 (Read & Write)  
- 사용자 참여 및 양방향 소통 가능  
- SNS, 블로그, 댓글 등 활발한 사용자 콘텐츠 생산

---

### 🔗 Web 3.0  
> 대표: D.Tube, Brave, BitTorrent, EOS

- **탈중앙화(Decentralization)**  
- 주요 기술: **블록체인**  
- 데이터를 특정 기업이 아닌 참여자들이 공동으로 소유  
- 위변조 방지를 위해 과반수의 검증 필요

#### 📦 관련 기술: NFT

- **Non-Fungible Token (대체 불가능한 토큰)**  
- 디지털 콘텐츠의 **원작자와 소유자 식별이 가능**
