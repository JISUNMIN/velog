함수형 프로그래밍(Functional Programming, FP)은 **"함수를 중심으로 코드를 작성하는 프로그래밍 방식"

함수를 수학 함수처럼 생각해서
입력이 같으면 항상 같은 결과를 내고,
함수 내부에서 외부 상태를 바꾸지 않는 걸 원칙으로 함.

즉, 부작용(Side Effect)을 없애고, 상태 변화를 최소화해서
코드를 더 예측 가능하고, 테스트하기 쉽고, 유지보수하기 좋게 만드는 프로그래밍 스타일

📌함수형: "물을 주는 함수와 얼음을 넣는 함수를 조합해서 새 물 함수를 만들어줘."

#### 왜 함수형 프로그래밍을 쓰는 걸까?
버그 줄이기
부작용이 없으니 코드가 예측 가능해지고, 디버깅 쉬움

병렬 처리에 유리
데이터가 변하지 않으니 여러 프로세스가 동시에 처리해도 안전

코드 재사용과 가독성 향상
작은 함수들을 조합해서 복잡한 기능을 구현 가능

자바스크립트에도 함수형 프로그래밍 스타일을 적용해서 탄생한 함수
📌map,filter,reduce,sort...


#### ES6+(ES2015 이후의 최신 문법) 과의 상관관계
직접적인 연관은 없지만 ES6+ 문법이 함수형 프로그래밍을 더 편하고 쉽게 사용 할 수 있게 해줌

1. 화살표 함수(=>): 함수형 프로그래밍은 **함수를 값처럼 다루기 때문에 **짧고 간결한 함수 정의 중요, 화살표 함수는 FP 스타일의 코드를 더 간결하게 만들어줌

```
// ES5
const doubled = numbers.map(function(n) {
  return n * 2;
});

// ES6+
const doubled = numbers.map(n => n * 2);
```

2. 스프레드 연산자(...)/구조 분해 :함수형 프로그래밍은 불변성을 중요하게 여김, 스프레드 연산자는 객체나 배열을 복사해서 상태를 변형하지 않고 새 값을 만들 수 있게 해줌

```
const user = { name: 'sun', age: 30 };
const updatedUser = { ...user, age: 31 }; // 원본 변경 X

```

✅ Q1. “함수를 값처럼 다룬다”는 건?
👉 자바스크립트에서는 함수도 값(value) 임
즉, 숫자, 문자열처럼 변수에 담을 수 있고, 인자로 넘기거나, 리턴값으로도 쓸 수 있다는 뜻

```
const greet = function(name) {
  return `Hello, ${name}`;
};

const sayHello = greet;        // ✅ 함수 자체를 변수에 할당
console.log(sayHello("Sun"));  // Hello, Sun

function callTwice(fn) {
  fn();
  fn();
}

callTwice(() => console.log("Hi")); // ✅ 함수를 인자로 전달

```

🎯 이게 왜 중요하냐면?
함수형 프로그래밍에서는 이런 방식으로  "함수를 조합하거나" "다른 함수에 전달하거나" "새로운 함수를 반환" 하는 식으로 함수를 다루는 구조를 많이 씀. 이게 바로 **고차 함수(Higher-order Function), 커링(Currying) 등의 기반** (고차 함수,커링이란? 참고)
즉, "함수도 숫자나 문자열처럼 마음껏 다룰 수 있다"는 게 핵심
