<p>영문화(i18n) 작업을 하다 보면 이런 상황을 자주 마주칩니다.</p>
<ul>
<li>문장 <strong>중간에 <code>*</code> 같은 표시를 넣고 싶음</strong></li>
<li>한 문장 안에 <strong>강조, 링크, 아이콘 등 다른 요소를 끼워 넣어야 함</strong></li>
<li>그런데 번역은 <strong>문장 단위로 깔끔하게 관리하고 싶음</strong></li>
</ul>
<p>특히 폼 UI를 만들다 보면<br />필수 입력 항목을 표시하기 위해 <code>*</code>를 붙이는 경우가 많은데,<br />이 <code>*</code>만 빨간색으로 표시하고 싶을 때 문제가 생길 수 있습니다.</p>
<hr />
<h2 id="기존-코드">기존 코드</h2>
<pre><code class="language-tsx">&lt;FlexDiv alignItems=&quot;center&quot; gap=&quot;5px&quot;&gt;
  &lt;b&gt;기타 비용 청구&lt;/b&gt;
  &lt;span style={{ color: &quot;red&quot; }}&gt;*&lt;/span&gt;
  은 필수 입력 항목입니다.
&lt;/FlexDiv&gt;</code></pre>
<p>기존 코드를 영문화(i18n)를 하려 하였습니다.
그런데 기존 코드와 같이</p>
<p><code>기타 비용 청구</code>와 <code>은 필수 입력 항목입니다.</code>를  
각각 따로 번역 키로 나누자니,</p>
<ul>
<li>영어에서는 문장 구조가 달라질 수 있고</li>
<li>번역 키도 의미 없이 쪼개지는 느낌이 들었습니다.</li>
</ul>
<p>그래서 문장을 <strong>하나의 번역 문장으로 합치고 싶었는데</strong>,  
이번에는 <strong><code>*</code>의 스타일링</strong>을 어떻게 처리해야 하나 생각이 들었습니다.</p>
<p><code>t()</code>로 문장 전체를 번역하면 결과가 문자열이라<br />문장 중간에 있는 <code>*</code>만 빨간색으로 처리할 방법이 없었기 때문입니다.</p>
<hr />
<h2 id="해결-방법">해결 방법</h2>
<p>&quot;문장은 번역으로 관리하면서, 
문장 중간에 스타일이 들어간 요소를 넣을 수는 없을까?” 하고 찾아보니<br /><code>react-i18next</code>에서 <strong><code>&lt;Trans /&gt;</code> 라는 컴포넌트를 지원한다는 걸 알게 됬습니다.</strong></p>
<p><code>&lt;Trans /&gt;</code>를 사용하면<br />번역 문자열 안에 <strong>React 컴포넌트를 직접 끼워 넣을 수 있습니다.</strong></p>
<p>즉,</p>
<ul>
<li>문장 전체는 하나의 번역 키로 관리하고</li>
<li><code>*</code>는 컴포넌트로 분리해서 스타일링할 수 있음</li>
</ul>
<hr />
<h2 id="trans--사용-방법"><code>&lt;Trans /&gt;</code> 사용 방법</h2>
<p><code>react-i18next</code>의 <code>&lt;Trans /&gt;</code> 컴포넌트를 사용하면<br />번역 문자열 안에 <strong>React 컴포넌트를 직접 삽입</strong>할 수 있습니다.</p>
<p>이를 통해</p>
<ul>
<li>문장 전체는 하나의 번역 키로 관리하고</li>
<li>문장 중간의 특정 요소(<code>*</code>, 강조, 링크 등)는 컴포넌트로 분리해</li>
<li>스타일과 구현은 코드에서 관리할 수 있습니다.</li>
</ul>
<hr />
<h3 id="컴포넌트-코드">컴포넌트 코드</h3>
<pre><code class="language-tsx">import { Trans } from &quot;react-i18next&quot;;

&lt;Trans
  i18nKey=&quot;기타 비용 청구 필수&quot;
  components={{
    b: &lt;b /&gt;,
    star: &lt;span style={{ color: &quot;red&quot; }}&gt;*&lt;/span&gt;,
  }}
/&gt;</code></pre>
<h3 id="번역-리소스">번역 리소스</h3>
<pre><code class="language-.json">// ko.json
{
  &quot;기타 비용 청구 필수&quot;: &quot;&lt;b&gt;기타 비용 청구&lt;/b&gt;&lt;star/&gt;은 필수 입력 항목입니다&quot;
}

// en.json
{
  &quot;기타 비용 청구 필수&quot;: &quot;&lt;b&gt;Additional cost claim&lt;/b&gt; &lt;star/&gt; is a required field&quot;
}</code></pre>
<p><code>components</code>에 정의한 <code>b</code>와 <code>star</code>는  
<strong>번역 리소스 안에서 사용할 커스텀 태그를 React 컴포넌트와 매핑하는 역할</strong>을 합니다.</p>
<p>번역 문자열 안에 작성된 다음과 같은 태그들은</p>
<ul>
<li><code>&lt;b&gt;...&lt;/b&gt;</code></li>
<li><code>&lt;star /&gt;</code></li>
</ul>
<p>실제 HTML 태그가 아니라,<br /><code>&lt;Trans /&gt;</code>가 해석하여 <strong>React 컴포넌트로 치환하는 플레이스홀더</strong>입니다.</p>
<pre><code class="language-tsx">&lt;Trans
  i18nKey=&quot;기타 비용 청구 필수&quot;
  components={{
    b: &lt;b /&gt;,
    star: &lt;span style={{ color: &quot;red&quot; }}&gt;*&lt;/span&gt;,
  }}
/&gt;</code></pre>
<p>위 코드 기준으로 매핑은 다음과 같이 이루어집니다.</p>
<ul>
<li>번역 문자열의 <code>&lt;b&gt;</code> → <code>components.b</code>에 정의된 <code>&lt;b /&gt;</code></li>
<li>번역 문자열의 <code>&lt;star /&gt;</code> → <code>components.star</code>에 정의된 <code>&lt;span /&gt;</code></li>
</ul>
<hr />
<h2 id="정리">정리</h2>
<ul>
<li>어떤 태그 이름을 사용할지는 <strong>번역 리소스에서 결정</strong></li>
<li>해당 태그가 실제로 어떤 React 컴포넌트인지는 <strong>코드에서 정의</strong></li>
</ul>
<p>이 구조 덕분에 번역 문자열에서는</p>
<ul>
<li>문장 구조와 강조 위치에만 집중할 수 있고</li>
<li>스타일링이나 구현 디테일은 컴포넌트 레벨에서 관리할 수 있습니다.</li>
</ul>
