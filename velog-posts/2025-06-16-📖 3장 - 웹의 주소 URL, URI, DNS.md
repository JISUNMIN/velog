## 📘 웹의 주소(URL)

- 웹 브라우저 상단의 주소창에 입력하는 웹페이지 주소나, 다른 페이지로 이동할 수 있게 구성된 링크 등을 **URL**이라고 합니다.
- 사용자가 웹에서 원하는 자원(페이지, 이미지, 영상 등)을 **정확히 찾아갈 수 있게 해주는 고유한 주소**입니다.
- 일반적으로는 웹사이트 주소 정도로 알고 있지만, 사실 **HTML 문서, 이미지, 동영상 등 인터넷에 존재하는 모든 자원을 나타낼 수 있습니다.**

> 우리가 한 덩어리로 알고 있던 URL은 사실 **프로토콜, 도메인, 경로, 포트번호, 파라미터** 등 여러 요소로 구성되어 있습니다.

---

## 🧱 URL 구조

주소를 쓸 때 시/군/구를 차례대로 쓰는 것처럼, URL도 일정한 **작성 규칙**이 있습니다.

예시:
> http://www.example.com:90/path/to/myfile.html?key1=value&key2=value2


| 구성 요소 | 설명 |
|-----------|------|
| `http:` | **프로토콜(통신 규약)**<br>- 예전에는 HTTP가 주로 쓰였지만, 요즘은 **보안이 강화된 HTTPS**를 더 많이 사용<br>- URL 왼쪽에 자물쇠 아이콘이 뜨면 HTTPS |
| `www.example.com` | **도메인(호스트 주소)**<br>- 우리가 접근하려는 서버 컴퓨터의 주소<br>- 실제로는 IP주소(`74.125.24.101`)지만, 도메인(사람이 읽기 쉽게 문자형태)로 표현 |
| `:90` | **포트 번호**<br>- 특정 자원에 접근하기 위해 사용하는 관문<br>- 기본 포트 번호는 생략 가능 (HTTP: 80, HTTPS: 443) |
| `/path/to/myfile.html` | **경로(Path)**<br>- 요청한 자원이 저장된 구체적인 위치<br>- 실제 물리 경로일 수도 있고, 가상의 경로일 수도 있음 |
| `?key1=value&key2=value2` | **파라미터(Parameter)**<br>- 서버에 전달하는 **추가 정보**<br>- `?` 뒤에 `key=value` 형태로 작성하며, 여러 개는 `&`로 연결 |

## 🔁 인코딩과 디코딩

### ✅ 인코딩(Encoding)

- URL에는 **영문자와 숫자를 제외한 대부분의 문자가 허용되지 않습니다.**
- 공백 등 허용되지 않는 문자는 기호로 변환되며, 이를 **인코딩**이라고 합니다.

예시:
- 공백 - `sweet potato` → `sweet+potato`  
- 한글 - `고구마` → `%EA%B3%...` (한글은 **백분율(%) + 16진수**로 인코딩)

### 🔓 디코딩(Decoding)

- 인코딩된 값을 **원래 문자로 되돌리는 과정**입니다.

## 🆔 URL vs URI vs URN
     URI
     > 🍪 URL
       🍪 URN

| 용어 | 의미 | 예시 |
|------|------|------|
| **URI**<br>(Uniform Resource Identifier) | 자원을 유일하게 식별할 수 있는 식별자 | 포괄적인 개념 |
| **URL**<br>(Uniform Resource Locator) | 자원의 위치를 나타냄 (주소) 예시: 책 위치 802번 서가 두번 째 줄 왼쪽에서 세번째| `https://google.com/search` |
| **URN**<br>(Uniform Resource Name) | 자원의 이름을 식별함 (고유 이름) 예시: 책의 고유 번호 ISBN 123-45-6780-012-3 | `urn:isbn:1234-5678` |

> URI는 **URL과 URN을 포함하는 상위 개념**입니다.

## 🌐 DNS (Domain Name System)

- **DNS는 도메인 주소를 IP주소로 변환**해주는 시스템입니다.
- 예시:  
  `www.google.com` → `123.456.789.1`
- 이처럼 사람이 기억하기 쉬운 주소를 컴퓨터가 이해할 수 있는 IP주소로 바꾸는 역할을 합니다.
- 실제 인터넷 환경에서는 전 세계에 **여러 대의 DNS 서버**가 있고, **케빈 베이컨의 6단계 법칙**처럼 서버 간 연결을 통해 빠르게 주소를 찾아냅니다.

**케빈 베이컨의 6단계 법칙**  
세상 모든 사람은 최대 6단계만 거치면 연결될 수 있다는 이론이에요.  
DNS도 이런 원리를 응용해 여러 서버를 거쳐 빠르게 주소를 찾습니다.

---

## ✅ 요약 정리

- **URL**: 웹 자원의 고유 주소로, 프로토콜·도메인·경로·포트·파라미터 등으로 구성
- **인코딩/디코딩**: URL에 허용되지 않는 문자를 기호로 변환하거나 다시 복원
- **URI**: URL과 URN을 포함하는 더 큰 개념
- **DNS**: 사람이 읽는 도메인을 컴퓨터가 이해하는 IP 주소로 변환해주는 시스템
